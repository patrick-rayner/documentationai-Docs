</Callout>

### Notes on flagged items

<ExpandableGroup>
  <Expandable title="Mint function" default-open="false">
    The mint function is required for staking emission. It is controlled through the **TokenMinter contract**, which maintains a whitelist of authorized minters. Only addresses on this whitelist can trigger minting, and the hard cap of 200M INKY is immutable in the contract. See [Contracts](/inky-token/contracts) for the full architecture.
  </Expandable>

  <Expandable title="Active ownership" default-open="false">
    Ownership is retained because the platform actively operates staking reward distribution. The owner address is the TokenMinter contract itself (not a personal wallet), and the creator wallet holds 0% of the supply.
  </Expandable>

  <Expandable title="LP tokens not locked" default-open="false">
    LP tokens are held by the **Products Vault** (99.997% of all LP tokens) rather than being locked in a third-party locker. See [Liquidity](/inky-token/liquidity) for details.
  </Expandable>
</ExpandableGroup>

## Independent verification

Anyone can verify the security posture of the INKY contract:

<Steps>
  <Step title="Review source code" icon="code" title-type="p">
    Open the [verified source code on BscScan](https://bscscan.com/address/0x75a320c97205dd2e70e09085d1408c73a73d4d8f#code).
  </Step>
  <Step title="Run automated scan" icon="shield" title-type="p">
    Run the contract address through [Token Sniffer](https://tokensniffer.com/) for an automated assessment.
  </Step>
  <Step title="Check ownership" icon="key" title-type="p">
    Check the `owner()` function to see which address holds admin privileges.
  </Step>
  <Step title="Read public parameters" icon="search" title-type="p">
    Review the `Read Contract` tab for all public parameters.
  </Step>
</Steps>
